Fase Cuatro: Evaluación del Modelo

Métricas de Desempeño

  • Accuracy: mide la proporción de predicciones correctas sobre el total.
  • Precision: mide cuántas de las predicciones positivas fueron realmente positivas.
  • Recall: indica cuántos de los casos positivos fueron correctamente detectados.
  • F1-Score: balance entre precision y recall, útil cuando hay clases desbalanceadas.
  • ROC-AUC: representa la capacidad del modelo para distinguir entre clases.

Validación Cruzada

Se aplicó validación cruzada k-fold (k=5), dividiendo los datos en varios subconjuntos para entrenar y evaluar el modelo múltiples veces. Con la validación cruzada, se reduce el riesgo de que los resultados sean extremos y se obtiene una medida más realista de qué tan bien funcionará el modelo con nuevos datos.

Funcionamiento:

  1. Se divide el conjunto de datos en 5 partes iguales (porque k = 5).
  2. El modelo se entrena 5 veces, cada vez usando 4 partes para entrenar y 1 parte diferente para probar.
  3. Cada parte se usa una vez como prueba y cuatro veces como entrenamiento.
  4. Se promedian los resultados para obtener una evaluación más precisa y confiable.

Análisis de Riesgos

  • Sobreajuste (overfitting): cuando el modelo aprende demasiado los datos de entrenamiento y no generaliza bien.
  • Subajuste (underfitting): cuando el modelo es demasiado simple y no logra capturar la complejidad de los datos.
  • Sesgos: en variables desbalanceadas, como el desequilibrio entre los casos de "Churn" y "No Churn".
