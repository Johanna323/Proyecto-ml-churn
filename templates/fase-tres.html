Fase Tres: Ingeniería del Modelo

Ingeniería del Modelo

Selección del Algoritmo

Se seleccionó regresión logística porque es un modelo simple, rápido y fácil de interpretar, además de ser muy efectivo para problemas de clasificación binaria como la predicción de pérdida de clientes (churn). Este algoritmo permite estimar la probabilidad de que un cliente cancele el servicio a partir de sus características demográficas, de uso y contractuales.

Validación

Para validar el rendimiento del modelo se dividió el conjunto de datos en:

  • 70% para entrenamiento
  • 30% para prueba

Entrenamiento

Se usaron datos ya procesados y transformados para entrenar un modelo de regresión logística en Python, asegurando que el entrenamiento tenga suficientes iteraciones para encontrar una buena solución.

Ajuste de Hiperparámetros

Como primer paso, se ajustó el parámetro max_iter=1000 para asegurar que el modelo converja sin errores durante el entrenamiento. Si el modelo no tiene suficientes iteraciones, podría quedarse a medio ciclo sin haber aprendido bien. Por eso, se aumenta max_iter a 1000, para darle tiempo de aprender correctamente.
